標題Re: [請益] 為什麼wiki概念下的條目不是一綱多本、多版併列？
時間Fri Jun 1 15:41:06 2012
※ 引述《lili141319 (lili)》之銘言： : 網路是一個多元而開放的環境 : 照理說針對同一個主題或條目 : 可能會有N個板本 : 不曉得為什維基百科 : （或類維基的網站如wiki-how） : 在同一個語言、同一個條目 : 卻基本上只有一個版本？ : 後來的編輯者 : 基本上不太可能大修本來的版本 : 又無法「另起爐灶」 : 這樣不就鼓勵「先搶先贏」？（雖然錯的還是會被修正） : 但如果是敘述方式的問題（你也不能說之前的版本錯 : 就像台北到高雄本來就有N種方法） : 為什麼不採取「一綱多本」「多版併列」的方式？ : （一綱「可能是」：立場中立、言之有據） : 而非得要整合出一個公定版（現存的形式） : →多版併列可以像是 : youtube你隨便打一個關鍵字， : 就有N多個影片（版本）供你選擇 : 然後再由閱聽者決定誰比較好（人氣、按like……） : 維基百科為什麼不是用這個模式？ wikipedia版有回過，這邊也回同樣： 從幾個角度來回答 1、最簡單的回答就是，這才叫「維基百科」（wikipedia），當然你也可以 用wiki程式，作一個一綱多本的百科，但那就不叫維基百科了。因為維基百科 的原則之一就是共同編輯，而不是「各自編輯」。 網路上每個人各自表述的管道多得是，像部落格、社群網站、微網誌、網路論壇 都是這樣，我想絕對不會差一個， 反而是像維基百科這樣能共同合作編輯內容的管道， 才是比較缺少的。 2、 那為何要共同編輯呢？比如 你說一綱的原則是「中立」，但不能把其他人的意見想法包括在裡面，非得自己 重新寫一套，這真的能稱得上是中立嗎？ 還是只是變成各說各話，沒有 任何交集？ 共同編輯的重點之一，就是不同意見的人彼此溝通、對話，找出 大家都可以接受的版本。（而且同個版本中，同樣可以納入不同的意見並陳） 3、從現實維護的角度來看，假如 每個人都可以寫自己的版本，例如假如 台灣這個條目有 108種版本， 即使是一個對台灣有興趣的人，會有人整天去管別人寫的107種版本中不中立？有沒有 錯誤？ 有沒有被破壞？寫得能不能看？ 不可能的。 這樣內容的品質就無法維持。 4、 你說的 有多個版本，由閱聽者決定那個比較好， 其實已經有類似的東西， 就是yahoo 知識+ ，由大家投票決定那個答案比較好，但，如果 有稍微用過就會感受到，明明某個答案是錯的還被選成對的 ，自己努力辛苦答的正確知識被排除的那種感覺， 知識的東西應該是用討論和理據，而不應該是「用投票」「用人數」來決定對錯。 5、你說的「多版本並列」，事實上也已經存在了，就是 百度、互動百科之中一些乏人管理的大雜彙條目，內容就是每個人貼一段， 每個人都不管別人寫的版本，只管貼上自己找來的內容，然後 然後整個條目亂成一團，然後讀者只好自己想辦法在一團雜亂中找到 自己要的內容。 如果想要多版本然後自己刪選內容，用google搜就好了，可以找到各種主題， 不同的網頁的介紹，但維基百科的編者，不止是丟資料給讀者， 而是要先刪選整理再呈現給讀者，這才是作為入門工具書的「百科」， 而不是搜尋引擎或資料庫。 6、現實而言，的確第一個編輯的人有某種優勢，但絕非絕對優勢， 實際上，通常寫新條目被審視的機會還比較多（被刪掉的機會也比較多） 而只要講出道理，後加資料的人，重新改變整個條目架構也是常有的 事（雖然比較費事） 而且，讓先寫的人有一點優勢有什麼不對？像中文版很多條目這麼多年來 都沒人寫，鼓勵大家多寫還沒有的內容不是很好嗎？ 7、 另一個實際的問題，wiki程式中有兩個種要的功能 「內部連結」、「跨語言連結」 前者讓文章中的特定字詞能連到該條目，後者能連結到其他語言的同個主題。 假如一個條目有50種版本，這連結要怎麼作？連結大家讓看到有50種版本然 後自己選一種？ 還要看運氣不要選到寫得特別差的？ 如果我是這個百科的讀者一定 煩死了。 -- ※ 發信站: 批踢踢實業坊(ptt.cc) ◆ From: 203.70.93.36
