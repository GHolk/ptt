因為 php 就和 html 混在一起的， html 本來就是很ㄏㄏ的語言， 只是標記語言，又不像 xml 那麼嚴緊， css box model 又很廢。 php 的函數又很沒格調， 各種奇怪的命名方式混在一起， 有 c 風格的 read write ； 又有直接讀整個檔案，效率也好很多的 file_get_content 。 其它一些奇奇怪怪的功能也包進去， 像 curl 、 exif ，結果一堆人只會用 php call curl ， 直接用 shell script 不是簡單多了嗎ㄏㄏ 不過以上主要是 php 要向下相容的結果， 就把一些古早時期的函數放到現在。 而那些函數創造時沒有深思熟慮。 然後還有一堆函數後來又加了幾個參數， 通常是 boolean 值改變行為，其實蠻煩人的。 php 發明人的理念也很沒格調： "I'm not a real programmer. I throw together things until it works then I move on." Rasmus Lerdorf 可以去看英文維基語錄，裡面一些記錄蠻好笑的； https://en.wikiquote.org/wiki/Rasmus_Lerdorf 程式可以單純是一種工具，只要能動就行。 然後寫 php 的菜逼八常常 html php css js 搞不清楚，都混在一起。 或一個函數寫幾百行， 架構整個噁心到不行，縮排亂七八糟， 還很多 sql injection 問題。 總之 php er 裡很多小白， 所以 php 給人印象一直不是很好。 大概是這樣。 ※ 引述《BetaGal (Betagal笑話,曲高和寡)》之銘言： : PHP : 一種程式語言 : 應用在很多 web 的 backend,ex巴哈姆特 : 最有名的framework應該還是laravel吧 : 之前常被靠北效能不好 : 但是前幾年php7出了之後似乎改善不少這種問題 : 但是不管職場還是網路上 : 人家聽到你寫php : 大概都會露出ㄏㄏ的表情 : 明顯的就是鄙視只會寫php的工程師 : 有沒有明明就很被廣泛使用 : 但是卻成為程式語言界的黑人的php的八卦? -- 8Ｈ(( ((88( ((((((((888(8((( ∫8段段??????? ﹊∴……﹛﹜ [m -- ※ 發信站: 批踢踢實業坊(ptt.cc), 來自: 140.116.102.187 ※ 文章網址: https://www.ptt.cc/bbs/Gossiping/M.1512271594.A.772.html
